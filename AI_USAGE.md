# AI Usage Documentation

## Overview
AI tools were used during development to assist with code structuring, debugging, and architectural clarity.  
All generated code was reviewed, modified, and validated manually before integration.

The goal was to use AI as an engineering assistant, not as a blind code generator.

---

## Where AI Was Used

### 1. Architecture Planning
AI assisted in:
- Designing backend–frontend separation
- Choosing Flask for lightweight API
- Selecting SQLite for simple relational storage
- Structuring REST endpoints

All decisions were reviewed and finalized manually.

---

### 2. Backend Development Assistance
AI helped generate:
- Initial Flask boilerplate
- Database schema suggestions
- API route structure

Manually reviewed and improved:
- Error handling
- Logic clarity
- Code readability
- Naming conventions

---

### 3. Frontend Assistance
AI supported:
- React component structuring
- Fetch API integration
- State management guidance

All UI logic tested and verified manually.

---

### 4. Debugging & Validation
AI used as debugging assistant for:
- Syntax errors
- API connection issues
- Data flow validation

All fixes were verified through manual testing.

---

## Constraints Followed

- No confidential or proprietary code used
- All code written specifically for this assessment
- AI-generated code reviewed before use
- Ensured correctness over blind generation
- Maintained readability and simplicity

---

## Engineering Philosophy

AI should:
- Assist thinking
- Speed development
- Improve clarity

AI should NOT:
- Replace understanding
- Generate unchecked code
- Create unnecessary complexity

This project reflects controlled, responsible AI-assisted development.
